# UI収束パス

構成・主操作/主対象・情報階層が全画面比較で合格した後だけ読む。構成不足や素材品質不足を数px調整で隠すために使わない。

## 1. 局所uplift / 構成再設計

| 判定 | 兆候 | 次の行動 |
|---|---|---|
| 局所uplift | 主役と読み順は明確。問題が1領域・1素材・1測定軸へ分離できる | Top1だけを1採否判断で動かす |
| 構成再設計 | 主役不在、同格要素、重複導線、過密と空白の同居、Top3が同じ構成原因 | 実装前に全画面モックを採用し、関連freezeを宣言して再オープンする |

「CTAを大きくする」を全画面へ一般化しない。メニューは主CTA、図鑑/一覧は選択対象、マップ/ゲームプレイはキャンバスやシーンが主役になり得る。

## 2. 採用モックの分解

spec/QAへ次を記録する。

- 採用モックのパス、ユーザー採用日、旧参照をsupersedeする範囲。
- 画面タイプと、縮小/ぼかしでも先に読む主操作または主対象。
- 存在する領域 / 存在しない領域。
- 主操作 / 補助操作 / 動的データ。
- 固定アンカー / 可変領域 / スクロール領域。
- 主要矩形と、素材/runtime/共通キットの分担。
- 変更するfreeze / 維持するfreeze。

## 3. 状態契約

全組合せの直積を作らず、代表状態と最も崩れやすい高リスク状態を選ぶ。

| 状態 | 固定seed/データ | 表示/非表示 | 固定アンカー | 可変領域 | 出力名 | smoke契約 |
|---|---|---|---|---|---|---|
| 代表 | 通常密度 | 通常要素 | 主操作/主対象 | 通常 | screen_default.png | 基本矩形・導線 |
| 高リスク | 0件、満杯、最長文、最大値、locked、optional非表示、modal等から1つ以上 | 差分要素 | 同じアンカー | 面積の受け手 | screen_state.png | 状態差・復帰 |

ユーザー提示スクショの問題を扱う場合は、その状態を必須行にする。before/afterはseed、表示データ、選択、viewportを一致させる。after/referenceでデータが一致しない場合は、読み順・密度・主役など比較する不変量を書く。

## 4. 空白の役割

空白を見つけたら、装飾やパネルを足す前に分類する。

| 種類 | 扱い |
|---|---|
| 状態変化で消えた既存スロット | 所有者が明確なら隣接する可変領域へ再配分候補 |
| 可変長コンテンツ/スクロールの予備 | 将来容量として維持 |
| empty stateの意味領域 | シェルを維持し、空状態メッセージ/導線で意味を与える |
| heroの呼吸 | 視線を休ませるため維持 |
| シーン/マップの意味空間 | 地理・航路・演出・プレイ可視域として維持 |

主要操作や反復操作を空白回収のために状態ごとへ移動しない。面積を渡す場合も、操作アンカーとパネル外形を先に固定する。

## 5. 収束順

1. **面積配分**: 原寸スクショで偶発的な空白を矩形として特定し、情報面積へ再配分する。
2. **主役階層**: 320×180程度の縮小/ぼかしで、画面タイプに応じた主操作または主対象が最初に読めるか確認する。
3. **状態間アンカー**: 代表/高リスク状態を重ね、主要操作・反復操作の意図しない移動をなくす。
4. **光学整列**: 構成freeze後かつ原寸で見える問題だけを扱う。単行見出し/数値群は描画画素中心またはbaseline、複数行本文は上端と行間で測る。内部間隔を保ってクラスタ単位で1軸だけ補正する。
5. **実使用寸法の素材契約**:
   - 9-slice/SVG: 最小幅/高さで罫線・飾りがcontent rectへ侵入しない。
   - 透明アート: alpha bboxと文字safe-areaが衝突しない。
   - 複合背景: ファイル全体でなく見えるwellを基準にし、runtime文字・hitbox・別viewportを同じ座標系で確認する。
   - ラインアイコン: stroke、tint、光学サイズが同役割で揃う。
6. **操作状態**: normal / hover / pressed / focus / selected / disabledのうち意味のある組合せを原寸で比較する。style resourceが存在するだけでなく視覚署名が異なることを確認する。
7. **原寸文字**: 最長文字・最大値・折り返しで省略、衝突、枠外漏れがないことを確認する。

## 6. Evidenceとsmoke

最低限残す。

- 同一状態の原寸before/after。
- after/採用参照の縮小比較。必要ならグレースケール。
- 代表状態と高リスク状態の原寸。
- 状態A→B→Aで可変矩形が元へ戻るsmoke。
- normalとの差があるinteraction style署名とフォーカス隣接。
- 撤去要素が存在しないこと、動的導線が残ることのsmoke。

独立レビューには結論を渡さず、採用参照/spec、原寸証拠、状態対応表、変更範囲を渡す。smoke greenだけで視覚合格にしない。

## 7. Stop / Revert

- 原寸全画面で第三者が改善を判別できなければrevertする。
- 光学補正は1回の収束パスへまとめ、各測定軸を微調整カウンタへ記録する。
- 同一軸を3回動かして改善しなければ、構成・文字設計・素材品質の別原因へ戻る。
- 意図した空白を埋める、主要アンカーを不安定にする、別状態を壊す変更は不採用にする。
