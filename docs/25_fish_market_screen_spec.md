# 25. 魚市場画面 仕様・参照分解

作成日: 2026-07-04
参照画像: `reference/10_fish_market_mockup.png`
対象: `src/ui/market_screen.gd`

## 1. 位置づけ

フェーズ6では、簡易実装だった魚市場画面をPNGメインUIへ正式化する。P5「調理費用」はMVPでは保留するため、本画面は「釣果を装備・船の資金へ換える」導線を主目的にする。料理は費用なしのまま維持し、売却時の注意文は「料理素材として残すか、装備資金に換えるか」に留める。

## 2. 参照画像の状態分解

`10_fish_market_mockup.png` は1画面内の通常売却選択状態を表す。ゲーム内では同じ `MarketScreen` 上で次の状態を切り替える。

| 状態 | 表示方法 | 備考 |
|---|---|---|
| 売却選択 | 左一覧・右査定・右下カートを表示 | 参照画像の主状態 |
| 売却確認 | 同画面上に確認オーバーレイ | 種類数・匹数・合計G、最後の1匹警告 |
| 売却完了 | カート領域のメッセージを更新 | 別画面へ遷移しない |
| 空状態 | 一覧を空メッセージへ差し替え | 全部売却後・再入場時 |

## 3. 存在する領域

| 領域 | 役割 | 実装方針 |
|---|---|---|
| 上部ステータス | Lv、装備竿、所持金 | 参照の三分割意匠は採用せず `PlayerStatusBar` に集約 |
| 左一覧 | クーラーボックス内の売却候補 | 7行表示。魚ポートレート、魚名、所持数、単価、数量操作 |
| 右上査定 | 選択魚の詳細 | 魚画像、レアリティ、売値、所持数、説明 |
| 右下カート | 売却予定と主操作 | 合計匹数・種類数・合計金額、全部選択、まとめて売る |
| 戻る導線 | 港へ戻る | 左下の戻るボタン |
| 確認オーバーレイ | 売却確定前の安全装置 | 同画面内の暗幕＋ダイアログ |

## 4. 存在しない領域・採用しない参照要素

- 別の売却結果画面は作らない。売却完了は同画面内の状態表示で完結する。
- 参照画像の上部三分割ステータスはそのまま再現しない。docs/19 の情報重複禁止と共通キット方針に従い `PlayerStatusBar` を使う。
- 参照画像にある魚の描き込みは実装用PNGには焼き込まない。魚は `FightFishAssets.card_portrait_path()` 経由でruntime表示する。
- 日本語テキスト、魚名、価格、数量はPNGへ焼き込まない。
- 行ごとの金縁フレームは使わない。行は静かな紙面ストリップと選択ハイライトで表現する。
- 調理費用・`cook_cost`・資金不足で料理不可の表示はMVP対象外。

## 5. 主操作と補助操作

主操作は `まとめて売る`。各行の `-` / `+` / `全部` は売却カートへ数量を入れる補助操作として扱う。`1匹売る` は専用ボタンを復活させず、数量1を選んで `まとめて売る` で統合する。

## 6. PNG / runtime 分担

| 種別 | 担当 |
|---|---|
| PNG | 市場背景、上部・左・右・下部のパネル質感、空の魚サムネ枠、空のボタン枠、装飾アイコン |
| runtime | 魚画像、魚名、レアリティ、所持数、単価、数量、合計金額、ボタン文言、警告・結果メッセージ |

## 7. smoke観点

- 旧 `ItemList` が残っていない。
- 7行の売却候補が表示され、数量が `0..所持数` に収まる。
- `+` / `-` / `全部` / `全部選択` / `まとめて売る` が正しく連動する。
- 複数魚種の一括売却で在庫と所持金が一度に更新される。
- 最後の1匹を売る場合、確認オーバーレイに警告が出る。
- 全部売却後、空状態が破綻しない。
- 港へ戻る導線が `harbor` へ遷移する。
