## 7. 判断ログ（直近パスのみ）

2026-07-09 出港プランカード ヒント再編を採用。

変更したもの:
- 餌魚行（行ラベル＋「餌魚を選ぶ」ボタン）と route payload の `shark_lure_fish_id` 送出を撤去（餌魚選択はREADYセレクタへ一本化。docs/38 §6を先に改訂）。
- ヒントを2行へ拡大: 1行目「本日の狙い目」（依頼対象魚→時間帯ブースト魚→未捕獲魚→固定文の決定的アルゴリズム）、2行目に既存の目撃談。固定チュートリアル文「釣る→売る／料理する→強化」は削除（低Lv向け動的ガイド化はdocs/43 §3の構想へ）。
- 時間帯行を旧餌魚行位置（0.530〜0.685）へ移動し、食事効果行はバフありのときのみ表示。
- `tools/harbor_screen_smoke.gd` を餌魚検証からヒント非空・食事効果行の表示切替・時間帯切替でのヒント再計算の検証へ差し替え。

変更していないもの:
- READYセレクタ側（`fishing_spot_select_screen.gd` / `fishing_screen.gd` / `fight_hud.gd`）の payload・`trip_stats` フォールバック処理。
- メガロドン前兆・目撃談（ヌシhint）のロジック、時間帯セグメント・グレード、右施設メニュー、フッター。

判断根拠:
- `docs/qa/evidence/harbor/2026-07-09_departure_plan_hint_compare.png`（3時間帯でヒント文言が変化）
- `docs/qa/evidence/harbor/2026-07-09_departure_plan_hint_asa_mazume.png`（原寸: 2行ヒント判読可・餌魚行なし・食事効果バフ表示）

採用理由:
- 危険海域でしか効かない餌魚選択が港から消え、初心者が押しても意味の分からない行がなくなった。
- ヒントが出現重みの実データと連動し、時間帯切替の効果が文言で体感できる。
