# ゲームデザイン仕様

## 1. 基本ループ

```text
港で準備
  ↓
狙う魚に合わせて釣り場ポイントを選ぶ
  ↓
釣り場で魚を釣る
  ↓
港へ持ち帰る
  ├─ 市場で売る → 装備資金
  └─ 調理して食べる → 食経験値＋次回バフ
  ↓
レベル・装備を強化
  ↓
より強い魚、港のぬしへ挑戦
```

## 2. 釣りファイトの判断

### 巻くべき状況

- 魚が休んでいる
- テンションが安全域の中央付近
- プレイヤー体力が十分にある

### 糸を出すべき状況

- 魚が突進、潜水している
- テンションが安全域上限へ近づく
- プレイヤー体力を回復したい

### 失敗条件

- テンションが切断限界を超える
- テンション5.5%未満が1.25秒以上続く
- 食いつき後、アワセ制限時間を過ぎる

## 3. 能力値

| 能力 | 効果 |
|---|---|
| 最大体力 | 巻き操作を継続できる時間が伸びる |
| 巻力 | 距離と魚体力を減らす効率が上がる |
| 技量 | 安全テンション域が広がる |
| 集中力 | アワセ可能時間がわずかに伸びる |

## 4. レベル設計

- 最大レベル：10
- レベルアップ資源：料理で得る食経験値
- 同じ料理の反復より、魚種×調理法の初回ボーナスを重視
- レベル帯ごとに新しい魚と釣り場ポイントを解放する
- レベルだけで自動勝利にはしない
- 低レベルでも上手ければ勝て、高レベルほど判断猶予が増える

## 5. 釣り場ポイント設計

釣り場ポイントは、プレイヤーが狙う魚を決めるための選択肢として扱う。

- 魚ごとに `min_level` と基本出現weightを持つ。
- 釣り場ポイントごとに、特定魚の出現weightへ補正をかける。
- Lv.1〜2帯の魚は多くの通常ポイントで釣れ、料理素材として集めやすくする。
- 高レベル帯の魚は対応ポイントで狙いやすくするが、補正後も低確率に抑える。
- ぬしは通常魚抽選から外し、専用ポイントまたは専用挑戦として扱う。

初期の釣り場ポイントは、カード式の選択画面で実装する。地図アートは後続の品質向上で追加する。

詳細は `docs/15_fishing_spot_encounter_spec.md` を正とする。

## 6. 料理設計

| 料理 | 解放 | 経験倍率 | 次回釣行効果 |
|---|---:|---:|---|
| 塩焼き | Lv.1 | 1.00 | 最大体力+5% |
| 刺身 | Lv.2 | 1.20 | アワセ猶予+0.35秒 |
| 煮付け | Lv.3 | 1.35 | 安全テンション域+5% |
| つみれ汁 | Lv.4 | 1.50 | 体力回復+18% |
| 魚フライ | Lv.5 | 1.60 | 巻力+10% |

### 食経験値式

```text
基本食経験値 = 魚の食経験値 × 料理倍率
最終食経験値 = 基本食経験値 + 初回料理ボーナス
初回料理ボーナス = 基本食経験値
```

初回判定は「魚種×調理法」の組み合わせで行う。

## 7. 経済設計

魚は二つの用途で競合する。

- **売却**：即時に装備を強化できる
- **料理**：長期的にプレイヤーを強化できる

ぬし攻略にはレベルと装備の両方が有効なため、片方だけを選び続けるより、状況に応じて配分する遊びを狙う。

## 8. ぬし設計

### 港のぬし・大岩クロダイ

- 解放：Lv.5
- 推奨：Lv.7以上＋磯竿以上
- 特徴：通常魚より突進頻度、潜水量、テンション急上昇が大きい
- 腕前が高ければLv.5でも攻略可能
- 料理バフと装備を用意すれば安定性が高まる
