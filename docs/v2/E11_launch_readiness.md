## E11-1. 設定画面（新画面）

- 新画面 `src/ui/settings_screen.gd`「せってい」。導線はタイトル画面＋港画面メニュー
- 項目（最小構成）:
  1. **BGM音量 / SE音量**（スライダー。`AudioServer` のバス操作。現状バス未分離のため `Master` 直下に `BGM` / `SE` バスを新設し、既存の再生箇所をバス指定へ寄せる）
  2. **フルスクリーン切替**（`DisplayServer.window_set_mode`）
  3. **セーブデータの削除**（二重確認ダイアログ必須。誤タップで消えない導線に）
- 設定の保存は `user://settings.json`（セーブ本体と分離。破損してもゲーム進行に影響させない）
- 子供向けUIなので文字は大きく、項目は増やさない。キーコンフィグはやらない（§E11-7 の入力方針決定に従う）
- 新画面のため `skills/ui-screen-build/` 工程。ただし装飾は既存共通素材の流用を第一候補にし、専用素材は最小限

## E11-2. セーブの販売品質化

1. **セーブ失敗のUI通知**: `save_game()` の失敗（tmp書き込み失敗・rename失敗）を `push_warning` だけでなく画面トーストで通知する。`PlayerProgress` に `save_failed(message)` シグナルを新設し、`ScreenBase` が受けて共通トーストを表示（全画面で機能する）
2. **セーブスロット**（§E11-7 の決定待ち。導入する場合）: `user://slots/<n>/tsuri_quest_save.json` 構造へ移行し、タイトルに簡易スロット選択（Lv・プレイ時間・最終日時を表示）。既存セーブは slot 1 へ自動移行。**E3のスキーマ変更と同時に実施**
3. チート対策は**しない**（シングルプレイヤー・子供向け。JSON平文を許容する方針を明記）

## E11-3. 表示・解像度

- §E11-7 の方針決定に従い、(a) `stretch/aspect="expand"` 維持＋16:10 / 4:3 の実スクショQAを visual QA に追加、または (b) `"keep"`＋黒帯へ変更のどちらかを実施
- 全画面スクショ比較は既存 visual QA スクリプトに解像度パラメータを足して流用する
- `config/name` から「MVP」を外し正式名へ。`assets/icon.svg` を専用アイコンへ差し替え、ブートスプラッシュを設定（差し替え素材は docs/31 台帳へ記入）

## E11-4. 入力

- §E11-7 の決定に従う。ゲームパッド対応する場合: input map（`project.godot` の `[input]`）へアクション定義を移し、キーボード直書き箇所をアクション参照へ置換。**対応しない場合もストアページ表記のために「マウス＋キーボード専用」を明示**
- 釣行中の中断（ウィンドウを閉じる）は現行の「ファイト中の魚を失うだけ・進行は直前セーブ済み」を仕様として明文化（対策コード不要）
