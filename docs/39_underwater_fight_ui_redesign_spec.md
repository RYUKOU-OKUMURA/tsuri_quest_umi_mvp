## 1. 参照画像（reference/14）の分解

### 存在する領域

| 領域 | 役割 | 実装方針 |
|---|---|---|
| 上部ステータス列 | 時刻・天候・所持金・釣り場/水深 | **現行 `FightStatusBar`（76px）を据え置き**。freeze維持 |
| 中央水中シーン | ファイトの主役。画面の約7割 | 現行 `UnderwaterView` を拡大表示（サイドバー廃止と下段スリム化で得た領域を充てる） |
| シーン右上フローティングカード | 魚名・レア度バッジ・推定サイズ・行動ヒント1行 | 新設。羊皮紙小型カード（約280×120px目安）。正体判明前は「未確認の魚影」表示 |
| 下段スリムバー | テンション／主操作／魚の体力 | 現行HUD（224px）を約140pxへスリム化。3ゾーン構成（§3） |

### 存在しない領域（削除するもの）

- 右サイドバー（幅326px）全体。魚カード・今の状況カード・タックル欄を含む
- 下段の「使用中のエサ」パネルと「操作のヒント」パネル（FIGHT中。キーチップは主操作ボタンに統合）
- FIGHT中の「釣り場変更」「港へ戻る」ボタン（Escの離脱オーバーレイへ集約。ファイト中の誤操作防止を兼ねる）

### 参照に在るが採用しない/調整する要素

| 要素 | 判断 |
|---|---|
| 参照画像内の文字入りパネル | すべてGodot runtime描画。日本語テキストのPNG焼き込み禁止 |
| 水中シーンの画風・魚の描き込み | 本docの範囲外。現行の背景・魚素材（`FightFishAssets`）を使う |
| カード内の行動アイコン | v1はrruntime描画の既存アイコン流用。専用PNGはv2候補 |

## 2. スコープ

### 対象

- FIGHT状態の下段バー・フローティングカード・サイドバー廃止
- 中間状態（CASTING / WAITING / APPROACH / BITE）の下段バー: 同じスリムバー構造で、主操作＝アワセ（E / Enter）を大きく出す。APPROACH/BITEの餌魚主語文言（docs/36改修C）はフローティングカードの行動行が引き継ぐ
- READYは docs/38 が正本（本docの対象外。ただしバーの枠素材・高さはREADY/FIGHTで共通化する）

### 対象外（やらない）

- 釣果ファンファーレ・結果オーバーレイ・離脱オーバーレイの変更
- 水面/水中ビューの背景・演出素材の変更（天候オーバーレイ・魚影のfreezeに触らない）
- 上部 `FightStatusBar` の変更
- fishing_simulator のロジック変更（表示専用の作業）

## 3. 下段スリムバー仕様（FIGHT）

高さ約140px（実測で±10px調整可。freeze新設は実装後）。3ゾーン:

| ゾーン | 内容 |
|---|---|
| 左 | テンションゲージ。「テンション」ラベル＋ハートアイコン＋分割バー＋「ゆるい/きつい」。分割数・色は現行を踏襲 |
| 中央 | 主操作2ボタン `巻く`（Space）/ `糸を出す`（Shift）。金縁大型ボタン＋キーチップ。バー上辺中央に小型の紺チップで `タナ 18.6m` |
| 右 | 魚の体力ゲージ。「魚の体力」ラベル＋魚アイコン＋分割バー＋「弱い/強い」 |

- 操作はキーボード（現行どおり長押し）とボタンのマウス押下の両対応。ボタンは押下中のみアクティブ（現行の `reel_changed` / `give_line_changed` シグナルを踏襲）
- タナの上下矢印インジケータ（現行HUDの▲▼）は、タナチップ内の小矢印として存続させるか実スクショで判断

## 4. フローティングカード仕様

- 位置: 水中シーン右上（シーンに重ねる。シーン枠の内側）
- 未確認時（アワセ前・正体秘匿）: 「未確認の魚影」＋反応タグ＋行動行（餌魚主語文言を含む）。魚名・レア度・サイズは出さない
- 判明後（`fish_revealed`）: 魚名＋レア度バッジ（`RarityStyles`）＋推定サイズ＋行動行（突っ込み等、現行「魚の行動」の1行版）
- カードは常時1枚。情報を増やしたくなったら docs/19 §4.6 原則4に立ち返る

## 5. 消える情報の行き先

| 現サイドバーの情報 | 行き先 |
|---|---|
| 魚カード（名前・レア度・推定サイズ） | フローティングカードへ |
| 「今の状況」カード | フローティングカードの行動行へ統合 |
| 生態解説（習性・好むエサ） | **削除**（魚図鑑で閲覧可能。ファイト中は行動不能な情報） |
| タックル欄（ロッド・リール・ライン等） | **削除**（仕掛け選択・ステータス画面で閲覧可能） |
| 釣り場詳細（右上パネル。READYのみ表示） | docs/38 のREADY仕様に従う |

削除2件は情報の恒久喪失ではなく、他画面に既存の表示があることを実装時に確認する。

## 8. 検証方針

- FIGHT操作（巻く/糸を出す/テンション破断/キャッチ/逃走）が現行と同一挙動であること（表示専用リファクタの確認）
- 正体秘匿: アワセ前にフローティングカードへ魚名・レア度・サイズが出ないこと
- docs/36の餌魚主語文言・好物ファンファーレが表示先移行後も機能すること
- 全状態×天候（晴/雨/霧）のスクショで文字収まり・カード重なりを確認
- 通常釣り場・危険海域・ボス釣り場の3系統でsmokeを回す
