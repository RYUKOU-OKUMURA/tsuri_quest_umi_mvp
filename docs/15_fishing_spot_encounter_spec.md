# 釣り場ポイント・出現抽選仕様

## 1. 目的

30種の魚を、単なるランダム候補ではなく「どの魚を狙うために、どの釣り場へ行くか」という釣行計画の対象にする。

この仕様では、レベル到達で釣れる魚を段階的に増やし、釣り場ポイントで狙いを絞れるようにする。ただし、高レベル帯の釣り場へ行くだけでレア魚が大量に釣れる状態にはしない。

## 2. 採用方針

- 魚の解放条件は `min_level` で管理する。
- 魚の基本的な釣れやすさは `weight` で管理する。
- 釣り場ポイントは、魚の `weight` に補正をかける。
- Lv.1〜2帯の魚は多くの釣り場で釣れ、確率も高めに残す。
- 高レベル帯の魚は対象ポイントで抽選に入る、または出やすくなるが、補正後も低確率に抑える。
- ぬしは通常魚抽選から外し、専用ポイントまたは専用挑戦モードで扱う。

## 3. プレイヤー体験

プレイヤーに次の判断を持たせる。

- 料理素材を集めたいので、港内や砂浜で低レベル魚を安定して釣る。
- 特定の魚を狙うため、魚の生息域に合うポイントへ行く。
- レア魚を狙うため、高レベルポイントへ行くが、低レベル魚も混ざる前提で粘る。
- エサ、竿、料理バフ、釣り場を組み合わせて釣行計画を立てる。

## 6. 出現抽選

通常魚の抽選は次の順で行う。

1. 選択した釣り場がプレイヤーレベルで解放済みか確認する。
2. `boss == false` の魚だけを通常魚候補にする。
3. `fish.min_level <= player_level` の魚だけを候補にする。
4. 釣り場の生息域ルールにより、候補魚へ補正係数をかける。
5. 補正後weightの合計から加重抽選する。

基本式:

```text
final_weight = fish.weight * spot_modifier * bait_modifier * event_modifier
```

初期実装では `bait_modifier` と `event_modifier` は `1.0` とし、釣り場補正だけを導入する。

## 7. 補正ルール

釣り場補正は、レア魚の価値を壊さないために次の制約を持つ。

- Lv.1〜2魚は、ぬし専用を除く全通常ポイントで `0.6` 以上の補正を持てる。
- 釣り場の主対象魚は `1.5`〜`3.0` 程度の補正を持てる。
- レア魚や大物は、主対象ポイントでも補正を上げすぎない。
- ある魚の最終出現率が高すぎる場合は、補正ではなく確率上限で抑える。
- 高レベル魚は、無関係な低レベルポイントでは原則として抽選に入れない。

初期の確率上限目安:

| 魚カテゴリ | 1匹あたりの上限目安 |
|---|---:|
| コモン | 25〜35% |
| アンコモン | 12〜20% |
| レア中型 | 6〜12% |
| レア大型 | 3〜8% |
| 終盤大物 | 1〜5% |
| ぬし | 通常抽選対象外 |

## 8. UI仕様

### 釣り場選択画面

港の「釣り場へ向かう」から釣り場選択画面へ遷移する。

本番マップUIでは、出航先の指定は航路図上の釣り場ポイントを主導線にする。下部パネルには釣り場別カード一覧を置かず、釣り手帳、全体達成度、選択中ポイントのヒント、メニューだけを置く。釣り場を選ぶ操作はマップ上のポイントに一本化する。

右詳細パネルに表示する情報:

- 釣り場名
- 解放レベル
- 水深目安
- よく釣れる魚の名前またはシルエット
- レア魚の気配
- おすすめエサ
- 未解放時のロック表示

地図アート追加後の下部達成度は、解放済み釣り場に記録した捕獲済み魚種数を集計し、`達成度 caught/target` として表示する。対象魚種は当面 `featured_fish` を使い、将来の図鑑/調査仕様で専用の達成対象リストへ置き換えられるようにする。釣り場別の達成度は釣り手帳側に寄せ、マップ下部には8セル台帳を出さない。

### 水中ファイト画面

ヒット前は魚名を伏せる既存仕様を維持する。

ただし、ヒット前の未確認カードには次を表示できる。

- 選択中の釣り場
- タナ/水深
- 反応の強さ
- 使用中のエサ

魚名、魚画像、No.、レアリティ、推定サイズは `FIGHT` 開始まで表示しない。

## 9. データ構造案

`GameData` に `FISHING_SPOTS` を追加する。

```gdscript
const FISHING_SPOTS := {
	"harbor_pier": {
		"id": "harbor_pier",
		"name": "港内・堤防",
		"unlock_level": 1,
		"depth_range": Vector2(5.0, 10.0),
		"description": "足場が良く、小魚が多い入門ポイント。",
		"common_modifier": 1.0,
		"fish_weight_modifiers": {
			"aji": 1.6,
			"iwashi": 1.8,
			"shirogisu": 1.1,
			"bora": 1.2
		}
	}
}
```

抽選APIは次の形へ拡張する。

```gdscript
func roll_normal_fish(player_level: int, spot_id: String = "harbor_pier") -> Dictionary
```

不正な `spot_id` は `harbor_pier` にフォールバックする。

## 10. 実装ステップ

1. `GameData.FISHING_SPOTS` と `roll_normal_fish(player_level, spot_id)` を追加する。
2. 釣り場ごとの確率サンプルを出す検証ツールを作る。
3. 港からカード式の釣り場選択画面へ遷移させる。
4. `FishingScreen` が選択中の `spot_id` を受け取るようにする。
5. 水中ファイトの上部ステータスまたは右カードに選択中の釣り場を表示する。
6. レベルアップ演出で新しい釣り場解放を表示する。
7. 後続で地図アート、エサ補正、時間帯補正を追加する。

## 11. 検証

最低限、次を自動確認する。

- Lv.未到達の釣り場を選べない。
- Lv.未到達の魚はどの釣り場でも抽選されない。
- Lv.1〜2魚は、ぬし専用を除く通常ポイントで候補に残る。
- 高レベル魚は対応する高レベルポイント以外で量産されない。
- ぬしは通常魚抽選に混ざらない。
- 各ポイントで1万回程度の抽選サンプルを出し、出現率が設計上限を超えていない。

## 12. 今回実装のゴール

今回のゴールは、港から釣り場ポイントを選び、選んだ釣り場に応じて30種の魚の出現候補と出現率が変わる状態にすること。

豪華な釣り場マップ画面、地図アート、時間帯補正、エサ補正は今回のゴールに含めない。まずは「どこへ釣りに行くか」で抽選結果が変わるゲームロジックを成立させる。

## 13. 今回の完了条件

- 港の「釣り場へ向かう」から釣り場選択画面へ遷移できる。
- 解放済みの釣り場は選択でき、未解放の釣り場は必要Lvが分かる状態で選択できない。
- 選択した `spot_id` が水中ファイト画面へ渡り、通常魚抽選に使われる。
- 未到達レベルの魚は、どの釣り場でも通常抽選に入らない。
- ぬしは通常魚抽選に混ざらず、専用ポイントで扱う。
- 高レベル釣り場でもLv.1〜2帯の魚が一定確率で混ざり、レア魚だけが量産されない。
- 対象魚は対応する釣り場で出現率が上がる。
- 水中ファイトの上部ステータスまたは右カードで、選択中の釣り場が確認できる。
- 既存の魚名非表示、魚画像表示、港へ戻る確認、釣果記録の挙動を壊さない。
- 釣り場別の出現率監査ツールで、レベルゲートと分布条件を確認できる。

## 14. 後続タスク

- 釣り場マップ画面を本番アートで作り込む。現在のカード式選択はロジック確認用の初期UIとして扱い、後続では島・港・砂浜・岩礁・外洋の位置関係が分かるマップ上でポイントを選べるようにする。
- 水上キャスト画面のREADY状態に「釣り場を変える」導線を追加し、港へ戻らずに釣り場選択へ戻れるようにする。釣り中またはファイト中は既存の中断確認を出し、魚が逃げる可能性を明示する。
- 「港へ戻る」は釣行終了・施設移動、「釣り場を変える」は続けて釣るためのポイント変更として、UI上の役割を分ける。
- レベルアップ演出で、新しく解放された釣り場を明示する。
- エサ補正、時間帯補正、天候補正を出現抽選に接続する。
- 釣り場ごとの背景、BGM、魚影演出を差し替える。
