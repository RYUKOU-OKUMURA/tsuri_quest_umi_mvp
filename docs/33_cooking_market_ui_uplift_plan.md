## 0. 結論（先に読む）

**両画面とも「構成・情報契約・可読性」の工事は完了済みで、残る品質差の主因は runtime 調整では埋まらない一点物素材の欠落である。**

- 基準品質の水中ファイト等は、**AI生成ソース画像**（`tools/source_assets/*_source.png`）をPILで加工して素材化した。一方、調理・市場の素材はほぼ全てが**純PIL描画**（`tools/generate_cooking_showcase_assets.py` / `generate_fish_market_assets.py`）で作られている。
- docs/19 §3.4（2026-07-05 改訂7）は既にこの限界を明文化済み: 「一点物（背景・キャラ・料理・王冠/月桂樹・バーストFX等）のプロシージャル生成は禁止。AI画像生成経路必須」。調理QAログの残ギャップも全状態で「一点物素材/演出フェーズへ送る」で終わっており、微調整カウンタ・装飾パス上限にも到達済み。**ルール上も、次の一手は素材フェーズしか残っていない。**
- 魚市場はさらに構造問題がある: 画面の見た目のほぼ全て（背景・全パネル枠・氷トレー・左下矢印）が **1枚の `fish_market_backplate.png`（PIL製 1280x720）に焼き込まれており、部分差し替えができない**。素材品質を上げる前に、この1枚焼きをスロット分解する工事が要る。

したがって実装案の柱は次の3つ。

1. **AI画像生成パイプラインの標準化**（docs/12形式brief → 生成 → `tools/source_assets/` → 加工スクリプト → `assets/showcase/`）。E4サメ素材（`generate_shark_fish_assets.py`）で実績あり
2. **市場backplateのスロット分解**（構成freeze値は維持したまま、背景/枠/トレーを独立素材化）
3. **P1掃除 → 状態別の一点物素材差し替え**を、1フェーズ1スロットで段階実施

## 2. なぜ品質が頭打ちになったか（構造要因の整理）

1. **一点物をPILで描いた**: 王冠・月桂樹・メダル・バースト・クーラー/コイン/時計アート・市場backplate全体。docs/19 §3.4 が禁止した経路で、参照級の質感には原理的に届かない
2. **市場は受け皿がない**: 1枚焼きbackplateのため、良い背景・トレー素材を作っても差し込むスロットがない
3. **runtime工事は上限到達**: 調理は微調整カウンタ・装飾パス累計・「参照未到達だが前進」3フェーズ上限をすべて消化済み。これ以上のStyleBox積み増しはルール違反かつ効果なし
4. **様式の不統一**: 写真調の料理画像と手描きPIL部品とAI生成級の食事アートが混在し、1枚絵として見たときの世界観が割れる。参照は全面「タイル調ピクセルアート+暖色ランタン光」で統一されている

## 3. 実装案

### 3.0 前提工事: AI素材供給ラインの標準化（最初に1回だけ）

E4サメで実績のある経路を、UI素材用に標準化する。**ここが「今後のお手本」の本体。**

```
docs/12形式の発注仕様（サイズ・スタイルアンカー・禁止事項・比較基準）
  → AI画像生成（参照画像の該当領域をスタイル手本として添付）
  → tools/source_assets/<screen>/ にソース保存（再取得可能性の担保）
  → 加工スクリプト tools/generate_<screen>_assets.py に「ソース加工モード」を追加
     （切り出し・透過・§3.3パレット量子化・等倍/2倍の解像度規約・9-slice分割）
  → assets/showcase/<screen>/ へ出力
  → 同じコミットで docs/31_asset_ledger.md §4 に生成サービス・日付・規約を記入
```

様式規格（全発注briefの共通条件）:

- 参照と同じ「ピクセルアート、暖色ランタン光、濃紺 #13283f + 金 #ffe7a8 + 羊皮紙 #f3e8cd 系」（正本は `src/ui/palette.gd`）
- **日本語テキスト焼き込み禁止**（可変値・和文はすべてruntime）。`LEVEL UP!` のような**固定英字**は素材化可（可読性が質感に直結するため推奨）。`+60 EXP` の数値部はruntime、光背のみ素材
- 写真調・3D調は不採用。既存の写真調料理画像（例: 煮付け）は差し替え対象
- 透過はクロマキー方式（キー色は実測サンプリング）

### 3.3 発注仕様（brief）の書き方

素材1スロットにつき docs/12 形式で1枚書く。必須項目:

1. 出力サイズ（表示等倍or2倍）と用途スロット名
2. スタイルアンカー: 参照画像の該当領域クロップを添付（`reference/` は方向性の真実。ただし切り抜きをそのまま採用しない）
3. 構図指定と**禁止事項**（日本語文字・UI枠の焼き込み禁止、余白規定）
4. 採用基準: 「現行に全画面比較で明確に勝つ」+「参照との縮小サムネ比較で対象差分が縮む」の2条件（docs/19 §5-11）

### 3.4 各フェーズ共通のDoD

- `./tools/cooking_visual_qa.sh` / `./tools/market_visual_qa.sh` green（該当側）
- 対象にした差分が**縮小サムネ比較で縮んだと第三者に判別できる**（before勝ちだけで採用しない）
- `./tools/validate_project.sh` + 該当smoke（`cooking_verify.sh` / `market_smoke.tscn`、セーブに触れたら `save_system_verify.sh`）green
