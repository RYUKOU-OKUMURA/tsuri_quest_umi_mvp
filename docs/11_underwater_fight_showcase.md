# 11. 水中ファイト看板画面 実装仕様

## 目的

`reference/02_underwater_fight_mockup.png` を水中ファイト画面の品質基準にする。まずはクロダイ戦 1 画面だけを、背景・魚・UI 枠・ゲージ・演出が揃った看板画面として作り込む。

## 到達ライン

- 水中背景は `_draw()` の図形ではなく、専用ラスタ背景を敷く。
- 主役魚は楕円やポリゴンではなく、透明 PNG のスプライトシートで表示する。
- 釣り糸、エサ、ヒット演出、泡、魚影は背景と魚に重なるレイヤーとして扱う。
- 既存の釣りロジック、テンション、魚体力、深度、行動名は維持する。
- 本番アート到着時に、同じファイルスロットへ PNG を差し替えれば画面品質を上げられる構造にする。

## 現在素材の扱い

`tools/generate_underwater_showcase_assets.py` で生成した初期 PNG は、素材ベース表示の技術検証用であり、完成方向の土台ではない。現在の `underwater_battle_bg.png` は `tools/build_reference_underwater_background.py` で、`reference/02_underwater_fight_mockup.png` の水中窓全体から抽出したリファレンス由来の背景パスである。魚、ヒット演出、糸、ルアーが入る領域は広めにマスクして水色面で埋め、左右の岩場、海藻、泡、遠景魚、水面光、海底光は参照画像の実ピクセルを残す。中央の補完面には、参照画像の左右海底/水中ピクセル、低コントラストの泡、薄い水底光、参照水面光を伸ばした薄い光パッチ、下部だけに抑えた参照由来の水底/岩場断片、上からの光芒、遠景魚影、中央寄りの小泡、参照の左右魚群/水中ピクセルから作る低コントラストの魚群テクスチャ、安全な左右端の水域から作る明るめで低アルファのミッドウォーターパッチ、左右端の水域から作る低透過の水平帯崩し霞、少し上へ持ち上げた参照由来の海底棚、上部左右の安全水域から作る少し持ち上げた水面光グレーズ、左右端の海底ピクセルから作る中央サンドチャンネルと低アルファの中央床反射、subject mask 内だけにかかるクリアウォーター/床光グレーズを再合成し、丸い塗りつぶし跡と空白感を弱める。中央に暗い岩場断片が出すぎると参照より黒い帯に見えるため、reef fragment と暗い midwater patch は低アルファに抑える。補完パッチは直線の水平帯として読まれないよう、波状の縦フェードとソフトな不定形マスクでアルファを絞る。中央下部の参照ピクセルはヒット文字や糸の残像を含みやすいため、新規パッチは主役演出から離れた左右端クロップだけを使う。左右端クロップを結合して中央へ伸ばす場合は `_stitch_reference_patches()` で重なり部分をグラデーション合成し、単純連結による縦 seam を避ける。最新パスでは、ImageGenで作成した `tools/source_assets/underwater_center_paintover_candidate.png` を全置換ではなく中央補完用素材として使い、リファレンス由来の左右岩場を残したまま、柔らかい中央全体マスクと主役周辺の強めマスクで床光・中層明度・水中密度を合成する。さらに同じ候補画像を canvas scale の中央床光リフトとして重ね、魚/ヒット周辺に明るい床反射を戻す。後段の reference-light polish では、安全な上部/左右床クロップに加えて、広い中央光ゲート、上面 shimmer、独立した surface bloom、強めた床 caustic strokes/arcs を低アルファでラスタ合成し、魚/糸/ヒット領域を再サンプリングせずに表面光と水底密度を増やす。水中窓が横長で下部テクスチャを大きくクロップするため、visible-floor-density polish では安全な左右端の床ピクセル、小泡、細い水底光を中下部へ持ち上げて、実画面で見える魚下からヒット周辺の密度を足す。最新の横並び比較では、明確な黒い帯、マスク境界、破綻した切り抜き、魚/糸/ルアー/ヒット残像は見えず、魚・釣り糸・ターゲット線・距離表示も読めるため、背景は今回パスでは合格として固定する。最終理想との差分は残すが、背景の明るさ、泡、床光、中心密度だけを理由に追加修正しない。背景に触るのは、全画面 side-by-side QA で P1 の黒帯、硬いマスク境界、残像、または主役要素の読みにくさが再発した場合に限る。旧生成背景ソース `tools/source_assets/underwater_battle_bg_source.png` と `tools/enhance_underwater_battle_bg.py` は、リファレンス抽出が使えない場合の代替パスとして残す。`kurodai_showcase_sheet.png` はリファレンス魚の切り出しを背景抽出して作った高品質パスである。右カードでは泳ぎ用シートを直接描かず、紙背景に焼き込んだ `kurodai_card_portrait.png` を使う。`UnderwaterView` では主役魚の背後に大きなシアンの戦闘オーラを描かない。参照画像にない広い円形エフェクトは、背景素材の密度を潰してデバッグ合成のように見えるため、影は小さく薄い接地感に留める。`hit_burst.png` は `tools/process_underwater_fish_assets.py` で青い水しぶき素材として再生成し、暖色は Godot 側の「ヒット！」文字だけに寄せている。ヒットバッジは主役魚と文字を邪魔しないよう、白い放射線や泡粒を入れすぎない。画面全体の品質判定は引き続き `reference/02_underwater_fight_mockup.png` との横並び比較で行う。

`sidebar_frame.png`、`top_status_frame.png`、`fight_hud_frame.png` は、生成素材の金飾りが強すぎたため `tools/generate_underwater_ui_frame_assets.py` で参照画像寄りの紙カード/濃紺ゲージ台として作り直している。これは最終美術素材ではなく、文字・ゲージ・アイコンが破綻しない完成寄りの枠素材スロットである。`FishingScreen` では右サイドバー外側の汎用パネルを外し、専用 `sidebar_frame.png` が直接画面に出るようにしている。現行の生成ルールでは `_draw_clean_card()` を使い、黒い太枠・鋲・強い金線を減らして、参照画像の紙カード寄りに軽量化している。右パネル下段カードは、白い罫線付きフォームに見えないよう、一枚紙カード上に透明切り抜きアイコンと本文を印字する。本文ベースラインと重なる上側ガイド線は焼き込まず、残す装飾は下端の薄い罫線と弱い角飾りに留める。直近パスでは、弱めすぎて薄いフォームに見えた action/tackle の濃紺枠、金線、内側罫線を控えめに戻し、タックルカードを控えめな竿/リール切り抜きと5行・13px装備情報で読ませる。さらに本番カード素材フェーズの第1パスとして、右パネルの魚カード紙面と action/tackle 本文紙面だけに参照画像由来の紙テクスチャを混ぜ、固定済みレイアウトや行数を変えずにフラットなランダム紙感を弱める。最新の右パネル素材パスでは、`sidebar_frame.png` に限定して外周の暗い土台、下段濃紺カードの黒枠、薄いシアン水平線、角リベットを弱め、画面内で黒い機械箱に見える状態を少し抑える。加えて、右魚カードと action/tackle 本文紙面にだけ低アルファの方眼、短い擦れ、薄い紙染みを決定的に合成し、参照の印刷カードらしい紙面密度へ近づける。現在は `_reference_paper_texture()` に明示的な参照crop指定を持たせ、魚カード紙面、行動カード本文紙面、タックルカード本文紙面がそれぞれ対応する参照カード領域のぼかし紙テクスチャを使う。参照の文字や魚を直接焼き込まず、紙のトーンと微細なムラだけを持ち込む。`_draw_navy_card()` の lighter fill/line alpha と `_draw_sidebar_paper_detail()` は右パネル呼び出しだけに使い、受け入れ済みの `fight_hud_frame.png` や `top_status_frame.png` へ波及させない。重い機械フレームには戻さず、参照のJRPG情報カードとして境界と情報密度が見える状態を優先する。

`kurodai_showcase_sheet.png`、`kurodai_card_portrait.png`、`hit_burst.png`、`fight_lure.png`、`hud_bait_icon.png`、`hud_tension_icon.png`、`hud_stamina_icon.png`、`hud_key_a.png`、`hud_key_b.png`、`hud_key_lr.png`、`hud_key_plus.png`、`hud_key_minus.png` は、`tools/source_assets/kurodai_final_art_source.png` がある場合はそれを優先し、なければ `kurodai_chroma_source.png` と `reference/02_underwater_fight_mockup.png` を元に `tools/process_underwater_fish_assets.py` で生成する。最終魚アート入力は、単なるクロマキー技術検証ではなく、鱗・ヒレ・目・陰影が読める手描き寄りの本番魚素材として扱う。処理内容は、境界色サンプリングによるクロマキー除去、マゼンタ縁のデスピル、半透明縁の再クリーニング、単体魚または4フレーム素材の正規化、透明化で落ちた鱗/ヒレのコントラストを戻す控えめなシャープ/ディテール復元、上半身・ヒレ端・高輝度部を少し水中寄りに冷ますマテリアル調整、腹側の白浮きを抑える控えめな明度キャップ、参照に近い背びれ/尾びれ/頭部線を読ませる最終魚アート用の線締め、低アルファの明るい切り抜き縁の抑制、透明縁/上半身/ヒレ端だけに足す低アルファの水色馴染み、単体魚ソースから作る控えめな尾びれ側の4フレーム差分、実戦用シートだけに入れる薄い水中エッジ下敷き、右カード用の透明ポートレート生成、ヒットバッジ生成、参照モックアップからの透明なエビ/疑似餌素材抽出、下部HUD用の透明なオキアミアイコン、テンション用ハートアイコン、魚体力用の小魚アイコン、A/B/LR/+/-キーキャップ抽出である。`kurodai_showcase_sheet.png` のフレーム差分は頭・目・胴体の鱗を安定させ、尾びれと尾柄付近だけを少し動かす。4セルを完全な複製に戻すと実戦画面で静止画が貼られた印象になるため、同一素材からの小さなポーズ差分を保つ。`kurodai_card_portrait.png` は右パネルの実表示窓に合わせた 560x310 の透明キャンバスにし、横長すぎる素材や透明余白で魚の存在感を落とさない。カード内の魚はリファレンスの図鑑カードに合わせて左向きにし、紙背景や丸角窓は持たせず、透明切り抜きと薄い接地影だけにする。現行生成ではカード用ポートレートの実魚 alpha bbox を約480px幅まで広げて、右カード上のクロダイが参照より小さく見える問題を弱める。カード側の紙面、淡いグリッド、仕切り線は `sidebar_frame.png` が持つ。カード用ポートレートは、水中エッジ下敷きを入れる前のクリーンな切り抜きを使い、紙面上に青いハロが出ないようにする。`fight_lure.png` はコード描画の丸/針ではなく水中糸先の小さな暖色餌として表示し、青背景や魚の端を透明マスクで落とす。`hud_bait_icon.png` は下部HUDの紙カード上に載せる大きめのオキアミ絵として使い、水中ルアー素材を無理に流用しない。`hud_tension_icon.png` は下部HUDのテンション見出しに使う小さな参照由来ハートで、共通アイコンシートを赤く塗りつぶした代替表示へ戻さない。`hud_stamina_icon.png` は下部HUDの魚体力見出しに使う小さな参照由来魚アイコンで、共通アイコンシートを水色に塗った代替表示へ戻さない。`hud_key_a.png` / `hud_key_b.png` / `hud_key_lr.png` / `hud_key_plus.png` / `hud_key_minus.png` は下部HUDの操作ヒントと右メニューで使う参照由来キーキャップで、コード描画キーはこれらがない場合のフォールバックに限定する。生成元は内製差し替え用の中間素材で、画面表示は最終PNGのみを参照する。水中窓の枠は、多重インセット線を重ねると参照よりモニター/デバッグ画面に見えるため、showcase path では低アルファの外周1pxとごく薄い内側ハイライト1pxだけに留める。

水中ファイトの主要文字は `src/ui/fight_fonts.gd` から `MPLUS1p-Regular.ttf` / `MPLUS1p-Bold.ttf` を読み込む。通常テーマ全体とは別に、看板画面の上部ステータス、HUD、右パネル、ヒット文字だけ水中ファイト専用フォント経路へ寄せる。現行v1ではアンチエイリアスをOFFにして、滑らかな現代アプリ文字ではなく、参照のJRPGカードに近い硬めの印字として読ませる。ただし小さい説明やカード内の単位/ラベルまで太字にすると黒い帯やデバッグラベルに見えるため、右魚カードの推定サイズ行などは通常フォントの小ラベル/単位と太字の数値を分けて印字する。

次フェーズでは、コード生成の簡易素材を磨くのではなく、以下を満たす完成素材を作ってから組み込む。

- クロダイは魚らしいシルエット、鱗、ヒレ、目、背と腹の陰影が読める。
- 背景は岩場、海藻、泡、光芒、遠景魚、海底の奥行きを持つ。
- 下部 HUD と右パネルは、参照画像の情報カード品質に近い専用枠素材を使う。
- 金色の格子状・デバッグ風に見える UI スキンは使わない。

## レイヤー構成

1. `underwater_battle_bg.png`
   水面光、光芒、海底、岩、海藻、遠景魚、泡を含む 16:9 背景。
2. `underwater_color_grade.png`
   背景の上に重ねる透明PNG。外周の暗部、海底の締まり、水面光の帯を足し、背景PNGの均一な明るさを抑える。
3. `underwater_seabed_detail.png`
   背景下部と左右に重ねる透明PNG。岩場シルエット、海藻、サンゴ、水底の光線を足し、生成背景の滑らかさを補う。
4. `underwater_foreground_ambience.png`
   背景の上に重ねる透明PNG。泡柱、中央の小さな泡列、低アルファの中央水底光、上部の広い光束と水面 shimmer、遠景魚影、粒子を含み、主役魚を邪魔しない密度補助として扱う。
5. 動的深度表示
   現在深度と目盛り。背景上に半透明で重ねる。
6. 釣り糸・エサ
   既存ロジックの位置に合わせて動的描画する。
7. `kurodai_showcase_sheet.png`
   主役魚。4 フレーム構成で、遊泳・緊張・突進・疲労を切り替える。同一ソースを使う場合も完全な複製ではなく、尾びれ側だけの控えめなポーズ差分を入れる。
8. `kurodai_card_portrait.png`
   右魚カード専用の静止ポートレート。透明切り抜きとして紙カード上に重ね、カード上で暗い矩形や青いハロが出ないようにする。
9. `hit_burst.png`
   アワセ直後の青い水しぶき台座。`hit_badge_full.png` がない場合のフォールバックとして Godot 側の文字と組み合わせる。
10. `hit_badge_full.png`
   アワセ直後の一体型「ヒット！」看板。参照由来の文字を青い水しぶき台座に合成した透明PNGとして、短時間だけ魚の手前に表示する。
11. HUD / 情報 UI
   上部ステータスバー、下部ゲージ、右パネル、ボタンは既存 Control UI を維持しつつ、順次画像枠へ差し替える。
12. `sidebar_frame.png`
   右側の魚情報・行動・タックルカード用の縦長フレーム素材。文字、魚、行動表示は Godot 側で重ねる。
13. `top_status_frame.png`
   時計、天候、所持金、地点/水深の上部ステータスバー用フレーム素材。左ファイトカラム内にだけ敷き、文字と数値は Godot 側で重ねる。
14. `top_status_icon_sheet.png`
   上部ステータスバー専用の時計、太陽、風、コイン小型アイコン。共通アイコンを縮小すると潰れて読めないため、上部カードでは専用素材を使う。
15. `fight_hud_frame.png`
   下部操作盤の一体型フレーム素材。ゲージ色、現在位置、操作文字、エサ情報は Godot 側で重ねる。
16. `hud_key_a.png` / `hud_key_b.png` / `hud_key_lr.png` / `hud_key_plus.png` / `hud_key_minus.png`
   下部操作盤の操作ヒントと右メニューに使う透明キーキャップ素材。参照下部HUDから抽出し、Godot 側では固定スロットへ収めて描く。
17. `fight_icon_sheet.png`
   時計、天候、風、コイン、テンション、魚体力、エサ、行動、タックルの共通アイコンシート。3x3 グリッドで各 UI が同じ素材を参照する。
18. `fight_action_card_icon.png` / `fight_tackle_card_icon.png`
   右パネル下段カード専用の小型アイコン。参照画像の下段カードから行動/タックル絵を切り出し、接続した紙背景だけを透明化して使う。共通アイコンシートは参照ファイルがない場合のフォールバックにする。

## 実装方針

- 水中ファイトの標準QA更新は `./tools/fight_visual_qa.sh` を使う。このスクリプトは `/tmp/tsuri_sidebar_static_compare.png`、`/tmp/tsuri_hud_static_compare.png`、`/tmp/tsuri_top_status_static_compare.png`、`/tmp/tsuri_full_static_compare.png`、`/tmp/tsuri_fight_compare.png`、`/tmp/tsuri_fight_compare.html` を更新する。実ランタイムキャプチャを試す場合だけ `TSURI_FIGHT_RUNTIME_CAPTURE=1 ./tools/fight_visual_qa.sh` を使う。実キャプチャが環境都合で失敗した場合は古い `/tmp/tsuri_fishing_fight.png` を残さず、静的比較を正式なfallbackとして使う。
- `UnderwaterView` は上記 PNG が存在すれば素材版を優先する。
- PNG がない場合は既存の procedural 描画にフォールバックする。
- `UnderwaterView` の showcase テクスチャは、プロジェクト既定の NEAREST ではなく LINEAR で表示する。UI枠はチャンキーなままでよいが、水中背景・魚・ヒット演出はリファレンスのなめらかなイラスト感に近づけるため、縮小時の硬いピクセル段差を抑える。
- 魚スプライトの位置、向き、状態は `FishingSimulator.visual_position`、`visual_direction`、`action_name`、`fish_stamina_ratio()` から決める。
- `FightStatusBar` は `top_status_frame.png` を左ファイトカラム内に敷き、時計・天候・所持金・水深のテキストを Godot 側で重ねる。全幅ルート直下に置くと右パネルが下がり、参照画像の「左ステータスカードと右魚カードヘッダーが同じ上端に並ぶ」構成から外れるため、右パネルとは横並びの同階層にする。上部の時計/太陽/風/コインは `top_status_icon_sheet.png` を優先し、共通アイコンシートは専用素材がない場合のフォールバックにする。`top_status_icon_sheet.png` は参照画像から透明抽出した128pxセルの4分割シートで、低アルファのハロを整理し、縮小表示前のシャープは控えめにする。時計/太陽/コインは上部カードで潰れないようセル内の有効面積を少し広げ、風アイコンは21pxインライン表示で大きくなりすぎないよう現行スケールを保つ。上部カードは小さな説明ラベルを詰め込まず、アイコンと強い数値/状態文字を優先する。特に所持金はコインアイコンと金額だけで読ませ、ラベルが金額を圧迫しないようにする。時計は `AM` と時刻を近づけ、`AM` は16pxの濃い印字、時刻と所持金は24px前後の太字を基準にする。AMから時刻までは31px前後のオフセットにして、薄い注釈ではなく同じ時計カードの印字として読ませる。数字が下端に食い込む場合は数字を大きくするよりベースラインを上げてカード中央に収める。現行の runtime 基準では、上部スロット比率を 23.5%/30.0%/25.0%/21.5% にし、右端の地点/水深カードを狭くしすぎない。天候と風は21px/19px、地点タイトルは14px、地点カードの水深行は通常フォント16pxの `水深` と太字19pxの `18.6m` を分け、文字アウトラインは1pxを基準にする。地点カードは参照の上下2段ステータスに寄せ、地点タイトルのベースラインをカード高の約33%、水深行を約75%に置き、狭い濃紺カード内で上下が詰まりすぎないようにする。トップアイコンはカード高の約 0.64 かつ 38-44px の範囲で描き、アイコンだけが大きく浮かないようにする。インラインの風アイコンは21px前後に抑える。天候カードの `風 弱` は参照の緑文字に寄せ、黒/紺の小さな注釈に見えないようにする。天候カードの縦区切りは太陽アイコン直後ではなく、`快晴` と風アイコンの間に置く。上部 parchment カードには淡い丸台座を焼き込まず、アイコンそのものをカード内の記号として立たせる。`top_status_frame.png` の parchment スロットは、共通 `_draw_card()` ではなく上部専用の軽い紙カードレンダラで作り、黒い外枠、鋲状の角点、強い縦区切りを抑える。紙面は単純なランダム粒状にせず、`reference/02_underwater_fight_mockup.png` の紙領域から取った低アルファのぼかし輝度を混ぜ、さらに控えめな紙縁の焼け、小さなムラ、低アルファの短い擦れ線/角の染みを重ねて、フラットな白板ではなく印刷カードの質感に寄せる。最新パスでは素材側の上下ガターと硬い影を減らし、カード本体を縦に広げて、黒地に小さく浮いた箱ではなく上端に貼り込まれた紙札として読ませる。カード間は連続した一本のバーに見えないよう、素材側で暗いガターを残して独立カードとして読ませる。今回の上部ステータス素材フォローアップでは、スロット比率、文字座標、アイコン座標を変えず、紙端の茶色い patina、外枠/内枠、上下ハイライト、角金具だけを強めて、白い矩形カード感を減らす。さらに parchment カード内部の焼き込み矩形線と上ハイライトは弱め、紙面がフォーム欄ではなく一枚札として読めることを優先する。`top_status_frame.png` のカード比率を変える場合は、`FightStatusBar._slot_rects()` も同じ比率へ更新し、素材枠と文字座標を必ず同期する。通常キャプチャが取れない環境では、`tools/build_fight_top_status_static_compare.py` で `/tmp/tsuri_top_status_static_compare.png` を生成し、参照上部ステータスと実素材ベースの静的ステータスを横並び比較する。
- `FishingScreen` の水中ファイト時レイアウトは、通常アプリ画面より外側余白とカラム間隔を狭くする。参照画像はステータス、海中窓、HUD、右パネルが画面端近くまで詰まっており、黒い外側ガターが広いと看板画面の密度が落ちるため、root margin、body separation、left-column separation、sidebar separation は控えめにする。

## 品質ゲート

毎回 `/tmp/tsuri_fishing_fight.png` と `reference/02_underwater_fight_mockup.png` を横並びで見て、以下を判定する。

1. 密度：背景、魚、UI、演出に参照画像相当の描き込みがあるか。
2. 余白：水中表示、右パネル、下部 HUD の占有率が参照に近いか。
3. 色：水中の青緑、暗部、金縁、情報カードの明度が破綻していないか。
4. 魚の存在感：クロダイが画面主役として読めるか。
5. UI枠の質：JRPGウィンドウ/情報カードとして見えるか。格子状・仮テーマ・デバッグ風に見える場合は不合格。
6. 背景固定条件：今回パスでは背景を合格扱いにし、黒帯、硬いマスク境界、残像、主役要素の可読性低下が再発した場合だけ背景を再調整する。
