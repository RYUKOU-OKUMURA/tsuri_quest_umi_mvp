# 11. 水中ファイト看板画面 実装仕様

## 目的

`reference/02_underwater_fight_mockup.png` を水中ファイト画面の品質基準にする。まずはクロダイ戦 1 画面だけを、背景・魚・UI 枠・ゲージ・演出が揃った看板画面として作り込む。

## 到達ライン

- 水中背景は `_draw()` の図形ではなく、専用ラスタ背景を敷く。
- 主役魚は楕円やポリゴンではなく、透明 PNG のスプライトシートで表示する。
- 釣り糸、エサ、ヒット演出、泡、魚影は背景と魚に重なるレイヤーとして扱う。
- 既存の釣りロジック、テンション、魚体力、深度、行動名は維持する。
- 本番アート到着時に、同じファイルスロットへ PNG を差し替えれば画面品質を上げられる構造にする。

## 現在素材の扱い

`tools/generate_underwater_showcase_assets.py` で生成した初期 PNG は、素材ベース表示の技術検証用であり、完成方向の土台ではない。現在の `underwater_battle_bg.png` は `tools/build_reference_underwater_background.py` で、`reference/02_underwater_fight_mockup.png` の水中窓全体から抽出したリファレンス由来の背景パスである。魚、ヒット演出、糸、ルアーが入る領域は広めにマスクして水色面で埋め、左右の岩場、海藻、泡、遠景魚、水面光、海底光は参照画像の実ピクセルを残す。中央の補完面には、参照画像の左右海底/水中ピクセル、低コントラストの泡、薄い水底光、参照水面光を伸ばした薄い光パッチ、下部だけに抑えた参照由来の水底/岩場断片、上からの光芒、遠景魚影、中央寄りの小泡、参照の左右魚群/水中ピクセルから作る低コントラストの魚群テクスチャ、安全な左右端の水域から作る明るめで低アルファのミッドウォーターパッチ、左右端の水域から作る低透過の水平帯崩し霞、少し上へ持ち上げた参照由来の海底棚、上部左右の安全水域から作る少し持ち上げた水面光グレーズ、左右端の海底ピクセルから作る中央サンドチャンネルと低アルファの中央床反射、subject mask 内だけにかかるクリアウォーター/床光グレーズを再合成し、丸い塗りつぶし跡と空白感を弱める。中央に暗い岩場断片が出すぎると参照より黒い帯に見えるため、reef fragment と暗い midwater patch は低アルファに抑える。補完パッチは直線の水平帯として読まれないよう、波状の縦フェードとソフトな不定形マスクでアルファを絞る。中央下部の参照ピクセルはヒット文字や糸の残像を含みやすいため、新規パッチは主役演出から離れた左右端クロップだけを使う。左右端クロップを結合して中央へ伸ばす場合は `_stitch_reference_patches()` で重なり部分をグラデーション合成し、単純連結による縦 seam を避ける。最新パスでは、ImageGenで作成した `tools/source_assets/underwater_center_paintover_candidate.png` を全置換ではなく中央補完用素材として使い、リファレンス由来の左右岩場を残したまま、柔らかい中央全体マスクと主役周辺の強めマスクで床光・中層明度・水中密度を合成する。さらに同じ候補画像を canvas scale の中央床光リフトとして重ね、魚/ヒット周辺に明るい床反射を戻す。後段の reference-light polish では、安全な上部/左右床クロップに加えて、広い中央光ゲート、上面 shimmer、独立した surface bloom、強めた床 caustic strokes/arcs を低アルファでラスタ合成し、魚/糸/ヒット領域を再サンプリングせずに表面光と水底密度を増やす。水中窓が横長で下部テクスチャを大きくクロップするため、visible-floor-density polish では安全な左右端の床ピクセル、小泡、細い水底光を中下部へ持ち上げて、実画面で見える魚下からヒット周辺の密度を足す。最新の横並び比較では、明確な黒い帯、マスク境界、破綻した切り抜き、魚/糸/ルアー/ヒット残像は見えず、魚・釣り糸・ターゲット線・距離バーも読めるため、背景は今回パスでは合格として固定する。最終理想との差分は残すが、背景の明るさ、泡、床光、中心密度だけを理由に追加修正しない。背景に触るのは、全画面 side-by-side QA で P1 の黒帯、硬いマスク境界、残像、または主役要素の読みにくさが再発した場合に限る。旧生成背景ソース `tools/source_assets/underwater_battle_bg_source.png` と `tools/enhance_underwater_battle_bg.py` は、リファレンス抽出が使えない場合の代替パスとして残す。`kurodai_showcase_sheet.png` はリファレンス魚の切り出しを背景抽出して作った高品質パスである。右カードでは泳ぎ用シートを直接描かず、紙背景に焼き込んだ `kurodai_card_portrait.png` を使う。`UnderwaterView` では主役魚の背後に大きなシアンの戦闘オーラを描かない。参照画像にない広い円形エフェクトは、背景素材の密度を潰してデバッグ合成のように見えるため、影は小さく薄い接地感に留める。`hit_burst.png` は `tools/process_underwater_fish_assets.py` で青い水しぶき素材として再生成し、暖色は Godot 側の「ヒット！」文字だけに寄せている。ヒットバッジは主役魚と文字を邪魔しないよう、白い放射線や泡粒を入れすぎない。画面全体の品質判定は引き続き `reference/02_underwater_fight_mockup.png` との横並び比較で行う。

`sidebar_frame.png`、`top_status_frame.png`、`fight_hud_frame.png` は、生成素材の金飾りが強すぎたため `tools/generate_underwater_ui_frame_assets.py` で参照画像寄りの紙カード/濃紺ゲージ台として作り直している。これは最終美術素材ではなく、文字・ゲージ・アイコンが破綻しない完成寄りの枠素材スロットである。`FishingScreen` では右サイドバー外側の汎用パネルを外し、専用 `sidebar_frame.png` が直接画面に出るようにしている。現行の生成ルールでは `_draw_clean_card()` を使い、黒い太枠・鋲・強い金線を減らして、参照画像の紙カード寄りに軽量化している。右パネル下段カードは、白い罫線付きフォームに見えないよう、一枚紙カード上に透明切り抜きアイコンと本文を印字する。本文ベースラインと重なる上側ガイド線は焼き込まず、残す装飾は下端のごく薄い罫線と弱い角飾りに留める。

`kurodai_showcase_sheet.png`、`kurodai_card_portrait.png`、`hit_burst.png` は、`kurodai_chroma_source.png` を元に `tools/process_underwater_fish_assets.py` で生成する。処理内容は、クロマキー除去、マゼンタ縁のデスピル、半透明縁の再クリーニング、単体魚または4フレーム素材の正規化、実戦用シートだけに入れる薄い水中エッジ下敷き、右カード用の透明ポートレート生成、ヒットバッジ生成である。`kurodai_card_portrait.png` は右パネルの実表示窓に合わせた 620x330 比率にし、横長すぎる素材を縮小表示して魚の存在感を落とさない。カード内の魚はリファレンスの図鑑カードに合わせて左向きにし、紙背景や丸角窓は持たせず、透明切り抜きと薄い接地影だけにする。現行生成ではカード用ポートレートの最大高さ/幅を上げ、透明余白を左右約60pxまで抑えて、右カード上のクロダイが参照より小さく見える問題を弱める。カード側の紙面、淡いグリッド、仕切り線は `sidebar_frame.png` が持つ。カード用ポートレートは、水中エッジ下敷きを入れる前のクリーンな切り抜きを使い、紙面上に青いハロが出ないようにする。生成元は内製差し替え用の中間素材で、画面表示は最終PNGのみを参照する。

水中ファイトの主要文字は `src/ui/fight_fonts.gd` から `MPLUS1p-Bold.ttf` を使う。通常テーマ全体は既存の `MPLUS1p-Regular.ttf` を維持し、看板画面の上部ステータス、HUD、右パネル、ヒット文字だけ太い表示に寄せる。

次フェーズでは、コード生成の簡易素材を磨くのではなく、以下を満たす完成素材を作ってから組み込む。

- クロダイは魚らしいシルエット、鱗、ヒレ、目、背と腹の陰影が読める。
- 背景は岩場、海藻、泡、光芒、遠景魚、海底の奥行きを持つ。
- 下部 HUD と右パネルは、参照画像の情報カード品質に近い専用枠素材を使う。
- 金色の格子状・デバッグ風に見える UI スキンは使わない。

## レイヤー構成

1. `underwater_battle_bg.png`
   水面光、光芒、海底、岩、海藻、遠景魚、泡を含む 16:9 背景。
2. `underwater_color_grade.png`
   背景の上に重ねる透明PNG。外周の暗部、海底の締まり、水面光の帯を足し、背景PNGの均一な明るさを抑える。
3. `underwater_seabed_detail.png`
   背景下部と左右に重ねる透明PNG。岩場シルエット、海藻、サンゴ、水底の光線を足し、生成背景の滑らかさを補う。
4. `underwater_foreground_ambience.png`
   背景の上に重ねる透明PNG。泡柱、中央の小さな泡列、低アルファの中央水底光、上部の広い光束と水面 shimmer、遠景魚影、粒子を含み、主役魚を邪魔しない密度補助として扱う。
5. 動的深度表示
   現在深度と目盛り。背景上に半透明で重ねる。
6. 釣り糸・エサ
   既存ロジックの位置に合わせて動的描画する。
7. `kurodai_showcase_sheet.png`
   主役魚。4 フレーム構成で、遊泳・緊張・突進・疲労を切り替える。
8. `kurodai_card_portrait.png`
   右魚カード専用の静止ポートレート。紙背景へ魚を焼き込み、カード上で暗い矩形が出ないようにする。
9. `hit_burst.png`
   アワセ直後の「ヒット！」演出。短時間だけ魚の手前に表示する。
10. HUD / 情報 UI
   上部ステータスバー、下部ゲージ、右パネル、ボタンは既存 Control UI を維持しつつ、順次画像枠へ差し替える。
11. `sidebar_frame.png`
   右側の魚情報・行動・タックルカード用の縦長フレーム素材。文字、魚、行動表示は Godot 側で重ねる。
12. `top_status_frame.png`
   時計、天候、所持金、地点/水深の上部ステータスバー用フレーム素材。左ファイトカラム内にだけ敷き、文字と数値は Godot 側で重ねる。
13. `top_status_icon_sheet.png`
   上部ステータスバー専用の時計、太陽、風、コイン小型アイコン。共通アイコンを縮小すると潰れて読めないため、上部カードでは専用素材を使う。
14. `fight_hud_frame.png`
   下部操作盤の一体型フレーム素材。ゲージ色、現在位置、操作文字、エサ情報は Godot 側で重ねる。
15. `fight_icon_sheet.png`
   時計、天候、風、コイン、テンション、魚体力、エサ、行動、タックルの共通アイコンシート。3x3 グリッドで各 UI が同じ素材を参照する。
16. `fight_action_card_icon.png` / `fight_tackle_card_icon.png`
   右パネル下段カード専用の小型アイコン。参照画像の下段カードから行動/タックル絵を切り出し、接続した紙背景だけを透明化して使う。共通アイコンシートは参照ファイルがない場合のフォールバックにする。

## 実装方針

- `UnderwaterView` は上記 PNG が存在すれば素材版を優先する。
- PNG がない場合は既存の procedural 描画にフォールバックする。
- `UnderwaterView` の showcase テクスチャは、プロジェクト既定の NEAREST ではなく LINEAR で表示する。UI枠はチャンキーなままでよいが、水中背景・魚・ヒット演出はリファレンスのなめらかなイラスト感に近づけるため、縮小時の硬いピクセル段差を抑える。
- 魚スプライトの位置、向き、状態は `FishingSimulator.visual_position`、`visual_direction`、`action_name`、`fish_stamina_ratio()` から決める。

## 品質ゲート

毎回 `/tmp/tsuri_fishing_fight.png` と `reference/02_underwater_fight_mockup.png` を横並びで見て、以下を判定する。

1. 密度：背景、魚、UI、演出に参照画像相当の描き込みがあるか。
2. 余白：水中表示、右パネル、下部 HUD の占有率が参照に近いか。
3. 色：水中の青緑、暗部、金縁、情報カードの明度が破綻していないか。
4. 魚の存在感：クロダイが画面主役として読めるか。
5. UI枠の質：JRPGウィンドウ/情報カードとして見えるか。格子状・仮テーマ・デバッグ風に見える場合は不合格。
6. 背景固定条件：今回パスでは背景を合格扱いにし、黒帯、硬いマスク境界、残像、主役要素の可読性低下が再発した場合だけ背景を再調整する。

## 次に本番化する素材

背景は今回パスでは固定する。クロダイ本体は、マゼンタ縁の削減、低アルファ縁の整理、実戦用シートだけの薄い水中エッジ下敷きまで完了。次は、1) クロダイの subtle なフレーム差分/接地影を最終比較で必要な範囲だけ詰める、2) 右パネルのカード/タグ/罫線/余白/情報密度、3) 下部HUDとボタンの質感/文字/状態表現、4) 上部ステータスのアイコン/タイポグラフィの順に詰める。

1. `sidebar_frame.png` は右パネル幅拡張、魚カード用 `kurodai_card_portrait.png` 分離、魚カード見出しの直書きタイトル行化、紙面の内部構造追加、行動文の意味改行、行動/タックル下段カード専用アイコン追加、下段カード本文の短縮、余白調整まで完了。`kurodai_card_portrait.png` は 620x330 の比率を維持しつつ、紙背景込みポートレートから透明背景の左向き切り抜きに変更した。最新パスではカード用ポートレート生成の最大幅/高さを 0.96/0.93 へ上げ、実魚 bbox を 501x281 まで広げ、左右透明余白を約60pxへ抑えて右カードの魚存在感を戻した。魚窓の紙面、淡いグリッド、仕切り線は `sidebar_frame.png` 側へ寄せ、二重の丸角ポートレートパネルに見える問題を減らしている。魚カード見出しは `No.028` とタグの間で魚名を中央配置し、`レア` タグをハイライト/影付きの小プレートにして、フラットなラベル感を減らした。最新パスでは、見出しの焼き込み塗りプラークと runtime の重ねプラークを外し、薄い区切り線だけを残して紙カード上へ直接印字する構造にした。さらに魚カードをヘッダー下へ上げたまま、runtime/素材側を y=10.9%、h=46.6% に同期して、魚カードが下段カードを圧迫しすぎる状態を弱めた。行動/タックルは y=58.8%/79.8%、h=19.5%/17.8% に広げ、下段の紙カードと本文に少し余白を戻している。魚カード下部は、素材側の強いガイド線を外し、Godot側の `推定 44.2 cm` と生態メモが線に切られない状態へ調整済み。コンパクトカードの魚窓を少し低くし、推定値を20pxへ抑え、説明ブロックを下げて、実寸表示で推定値・説明区切り線・生態メモが重ならない安全余白を優先した。コンパクトカードの生態メモは、通常説明文1行と緑ドット付きの `好むエサ` / `主な生息域` 2行に分けている。右カード下部に収まるよう12px相当に抑え、すべてを同じ箇条書きとして扱う薄い情報密度から、参照に近い説明文+情報行の構造へ戻している。行動/タックル下段カードは、見出し帯を浅くし、紙本文ウェルを縦に広げ、アイコン占有と内部ガイド線を抑え、Godot側の二重本文パネルを外し、紙スロット上端/下端に干渉しないベースラインへ調整済み。最新パスでは、下段カードのアイコン枠/本文枠の分離をやめ、参照画像下段カードから切り出した専用アイコンを透明背景の切り抜き素材として再生成し、一枚の紙カード上にアイコンと本文が印字されている見え方へ寄せた。さらに本文と競合していた上側ガイド線を削除し、残る下端罫線と角飾りを弱め、action アイコンは 56px、tackle アイコンは 88x70px の表示枠まで広げつつ、本文が `港の入門竿` まで読める幅を維持している。行動本文は15pxへ抑え、長い `潜ろうとしている` をカード内表示では `潜る` へ圧縮して2行に収める。タックル本文は太い20px二行から、通常フォント13pxの `ロッド / ライン / ハリス / 針` 4行へ変更し、参照の道具カードに近い情報密度へ戻している。通常キャプチャが取れない環境では、`tools/build_fight_sidebar_static_compare.py` で実素材と runtime 座標を使った `/tmp/tsuri_sidebar_static_compare.png` を生成し、右パネル単体の参照比較を行う。次は全体比較で右パネルが再び浮く場合だけ小さく調整する。
2. `fight_hud_frame.png` の上段は深度プレート強化、暗色化、HUDアイコン縮小/低透過化、右ラベル余白調整、18分割メーター化、未充填セグメント表示とハイライト/影追加まで完了。下段はエサ/操作/メニューの紙タイトル帯、本文スロット、濃紺メニュー行、操作ヒントの3スロット化、A/B/LRキー配置の整列、キーキャップ/ラベルの拡大と低めの再配置、キーキャップの暗い外枠/金内枠/上ハイライト、操作ヒント幅の拡張、キー下の短い補足文追加まで完了。最新パスではエサ/メニュー比率を維持したまま、操作ヒントをベージュのフォーム枠から青い見出し帯付きの操作カードへ変更し、見出しを中央の明るい文字へ、本文を一枚紙ウェルへ、縦区切りを低アルファへ寄せた。A/B は小さめの丸い物理ボタン、L/R は短いピル、メニューはクリーム色の丸い +/- キャップを使い、`/tmp/tsuri_hud_static_compare.png` で参照下部HUDと比較できる。`FishingScreen` と `FightHud` の表示最小高は 224px に上げ済みで、2048x456 のフレーム比率をより参照の厚い操作盤に近づけている。水中窓は少し短くなるが、最新比較では主役魚/ヒット領域の看板性は保てている。次は最終比較でまだ機械的に見える場合、HUDフレーム素材そのものの比率と小文字の光学サイズを調整する。
3. 上部ステータスの地点カードは参照に合わせてアイコンなし中央寄せに修正済み。`top_status_frame.png` は紙カード内枠、角金具、濃紺地点カードの内装追加まで完了。上部アイコン群は `top_status_icon_sheet.png` に分離し、時計/太陽/風/コインが潰れない状態まで改善済み。最新パスでは `top_status_icon_sheet.png` を96pxセルから128pxセルへ上げ、低アルファのハロを整理し、抽出後に軽いシャープをかけてから保存する。`FightStatusBar` は左ファイトカラム内へ移動し、右パネルヘッダーが上端から始まる構造に修正済み。カード比率は時計/天候/所持金を広げ、地点カードを締める方向へ調整済み。文字ベースラインと数値の光学サイズも調整し、所持金は小ラベルを外して金額を大きく読ませる状態まで改善済み。最新パスでは `top_status_frame.png` と `FightStatusBar._slot_rects()` のスロット境界を同時に更新し、紙カード側の淡い丸アイコン台座を削除し、さらに紙テクスチャ強度、内側罫線、ハイライト、天候/所持金カードの縦区切りを弱めて、縮小時の罫線/格子ノイズを抑えた。さらに parchment スロットだけ上部専用の軽い紙カードレンダラへ切り替え、黒い外枠、鋲状の角点、強い縦区切りを弱めた。最新の素材パスでは、参照モックアップの紙領域からぼかした輝度ムラを低アルファで混ぜ、平坦なランダム粒状ではなく印刷カード寄りの紙面明暗に寄せた。ランタイム側では上部アイコンを少し大きくし、アイコンと文字の距離、`AM 08:47`、`快晴 | 風 弱` のまとまりを参照寄りに詰め、さらに主数字/水深値のサイズを少し下げてベースラインを上げ、カード下端に食い込まないようにした。画面外側の root margin と主要 container separation も詰め、黒い外側ガターを減らして参照の画面密度に近づけている。次に詰めるなら、通常キャプチャ復旧後の最終比較で上部のアイコン/文字アラインメントを見直す。
4. ヒット演出は白い放射線、泡粒、下端の重なりを調整済み。青白い線が背景ノイズに見えないよう、`hit_burst.png` 側の ray/glint 密度は控えめにしている。最新パスでは runtime の水しぶき台座と `ヒット！` 文字をさらに少し小さくし、距離バーをヒット演出より後ろかつ低透過にして、`ヒット！` が水平バーに切られず、魚/ルアーの主役クラスターを圧迫しない状態へ寄せた。最終 HUD/フォント調整後に比較だけ再確認する。
5. 泡、光粒、魚影は `underwater_foreground_ambience.png` と `UnderwaterView` の補助光粒として追加済み。背景の均一な明るさは `underwater_color_grade.png` で少し締め、海底/左右の密度は `underwater_seabed_detail.png` で補っている。`underwater_battle_bg.png` 本体は `tools/build_reference_underwater_background.py` でリファレンス水中窓全体から抽出し、主役魚/ヒット/糸/ルアー部分をマスク補完して再生成済み。左右岩場、海藻、泡、遠景魚、海底光は参照画像の実ピクセルを残すため、旧生成背景や右端だけのパッチより密度が高い。中央補完面には参照由来の左右海底/水中テクスチャ、低コントラスト泡、薄い水底光、参照水面光を伸ばした光パッチ、下部だけに抑えた水底/岩場断片、遠景魚影、中央寄りの小泡、上からの光芒、参照由来の魚群/水中テクスチャ、左右端水域由来の明るめで低アルファなミッドウォーターパッチ、同じ左右端水域由来の明るめで低透過な水平帯崩し霞、上部左右の安全な水域から作る少し上げた中央ミッドウォーター光グレーズ、少し上げた海底棚、左右端海底由来の中央サンドチャンネル、少し明るく上げた低アルファの中央床反射、左右端海底由来の下部微細ディテールを戻し、丸い塗り跡と空白感を弱めている。最新パスでは中央ミッドウォーターと水平帯崩しの暗いパッチをさらに明るくし、アルファを落としたうえで、左右端海底クロップと低アルファの床光ストロークから中央床反射メッシュを追加し、さらに subject mask 内の final paintover cleanup で中層明度、床テクスチャ、小泡、床光を足した。加えて `tools/source_assets/underwater_center_paintover_candidate.png` を柔らかい中央全体マスク、主役周辺マスク、canvas scale の中央床光リフトで合成し、魚の背後からヒット文字周辺までの中層明度、水底密度、床光をかなり上げた。さらに canvas scale の reference-light polish で、上部左右の安全水域から中央の水面光を、左右端海底から下部の床反射を再合成し、広い中央光ゲート、上面 shimmer、低アルファの床 caustic strokes/arcs を足している。元魚/糸を含みやすい左中域・右中央クロップを避け、左右端クロップ同士の結合は `_stitch_reference_patches()` でブレンドする。補完パッチは波状の縦フェードと不定形マスクでアルファを絞り、横長の塗り面や縦 seam に見える問題を抑えている。中央下部のクロップを使うとヒット文字や糸の残像が残るため、新規パッチは左右端クロップまたは主役演出から離れた上部水域に限定する。`underwater_foreground_ambience.png` には中央の小さな泡列と低アルファの水底光も足し、主役魚やヒット文字を邪魔せずに中央水域の空白感を少し減らしている。最新パスでは `underwater_color_grade.png` の上部水面光、光芒、shimmer strokes を強め、上端の暗幕を弱めたうえで、`UnderwaterView` 側の showcase テクスチャを `Vector2(0.5, 0.24)` の上寄せ cover で描き、color grade opacity を 0.10、seabed detail を 0.22 に下げた。これで参照由来背景を補助レイヤーで覆いすぎず、水中窓内へ水面光を残す状態へ寄せている。魚下シャドウも薄くして、参照にない楕円状の暗さと灰色の膜を抑えている。`UnderwaterView` は LINEAR フィルタで showcase テクスチャを描画し、背景の縮小時の硬さを抑える。背景は今回パスでは固定し、以後はクロダイ本体、右パネル、下部HUD、上部ステータスの品質差分を優先する。
