# 魚図鑑UI 実装ゴールと進め方

Date: 2026-07-01

対象予定画面: `src/ui/fish_book_screen.gd`

参照画像: `reference/07_fish_book_mockup.png`

このメモは、魚図鑑画面を既存の簡易ステータス画面内の一覧から、ゲーム本体の世界観に合う本番寄りUIへ引き上げるための実装ゴールを定義する。`docs/17_fishing_spot_map_process_lessons.md` の進め方を踏襲し、最初に完成イメージの構成を分解してから実装へ入る。

## 結論

魚図鑑は、`status_screen.gd` の一部として軽く表示する画面ではなく、独立した「釣り手帳」相当の画面として作る。採用する完成イメージは、左に魚カード一覧、右に選択中の魚詳細、下にフィルタ、上に木製ヘッダーを置く構成とする。

目指す品質は、釣り場マップ画面と同じく「仮UIではない80点以上の本番寄り画面」。最初から完全な最終アートを目指すのではなく、画面構成、主導線、情報階層、文字の収まり、素材の質感を完成イメージへ寄せる。

## 完成イメージに存在する要素

- 背景は港と海。UIの外側や隙間から、港の世界観が見える。
- 上部に大きな木製ヘッダーがある。
- ヘッダー中央に `魚図鑑` の大見出しがある。
- ヘッダー左に発見数がある。例: `発見済み 18/30`
- ヘッダー右にプレイヤー情報がある。例: Lv、所持金、装備中の竿。
- 左側に3列の魚カードグリッドがある。
- 発見済みカードには魚番号、魚名、魚ポートレート、レア度、釣果数、最大サイズがある。
- 未発見カードには暗い魚シルエット、`？`、鍵表示がある。
- 左グリッドの右端にスクロールバーがある。
- 右側に選択中の魚の詳細パネルがある。
- 詳細パネルには大きな魚ポートレート、魚番号、魚名、レア度、釣果数、最大サイズがある。
- 詳細パネルには `生息地`、`好物`、`行動` の説明行がある。
- 詳細パネル下部に `よく釣れる場所` の小さな釣り場カード列がある。
- 下部にフィルタボタンが並ぶ。例: `全魚`、`港内`、`砂浜`、`岩礁`、`沖`、`レア`
- 下部右に強い `港へ戻る` ボタンがある。

## 完成イメージに存在しない要素

- 既存 `status_screen.gd` のような、プレイヤー情報、魚図鑑、所持品が横並びになった汎用3ペイン構成。
- 魚図鑑と料理記録を同じスクロール領域へ混在させる構成。
- 汎用 `PanelContainer` だけで作った白っぽいカードUI。
- 大きな説明文だけの画面。
- Webダッシュボード風の表。
- タイトルだけが大きく、一覧や詳細が弱いランディングページ風構成。
- 画像素材なしのテキスト中心UI。

## 実装ゴール

実装完了時に、ゲーム内で図鑑画面を開くと次の状態になっていること。

- 港メニューまたはステータス画面から魚図鑑へ遷移できる。
- 図鑑画面は `reference/07_fish_book_mockup.png` と同じ読み順になっている。
- 魚データは `GameData.get_all_fish_ids()` と `GameData.get_fish()` から取得する。
- 釣果データは `PlayerProgress.caught_counts` と `PlayerProgress.best_sizes` から取得する。
- 発見済み魚と未発見魚の表示差が一目で分かる。
- 魚カードを選ぶと右詳細が更新される。
- 未発見魚を選んだ場合は、名前や詳細を伏せつつ、未発見状態として成立する。
- フィルタで表示対象を切り替えられる。
- `港へ戻る` で港画面へ戻れる。
- 1280x720相当の実スクショで、日本語テキストが見切れない。
- 新規PNG素材がheadless previewでも表示される。

## 情報設計

### 左グリッドのカード

発見済みカードに出す情報:

- `fish_no`
- `name`
- `rarity`
- 魚ポートレート
- `釣果 N匹`
- `最大 X.Xcm`

未発見カードに出す情報:

- `fish_no`
- `？？？？？`
- 暗いシルエット
- `未発見`
- 鍵アイコンまたは暗いプレート

カード選択時:

- 選択中カードは金縁または明るい発光で主役化する。
- 発見済みなら右詳細に完全情報を出す。
- 未発見なら右詳細も伏せ表示にする。

### 右詳細

発見済み魚に出す情報:

- `fish_no`
- `name`
- `rarity`
- 大きな魚ポートレート
- `釣果 N匹`
- `最大 X.Xcm`
- `生息地`: `habitat`
- `好物`: `preferred_bait`
- `行動`: `behavior`
- `よく釣れる場所`: その魚が出る釣り場、または代表釣り場

未発見魚に出す情報:

- `fish_no`
- `？？？？？`
- 暗いシルエット
- `まだ釣ったことがない魚`
- 生息地や好物は伏せるか、解放済み釣り場に応じた短いヒントだけにする。

### フィルタ

最初の実装では次のフィルタを目標にする。

- `全魚`: 全魚種
- `港内`: 港内・堤防系の魚
- `砂浜`: 砂地、浅瀬、かけあがり系の魚
- `岩礁`: 岩礁、消波ブロック、根魚系の魚
- `沖`: 港外、潮目、回遊魚系の魚
- `レア`: レア以上またはぬし

内部的には、最初は `habitat` 文字列と釣り場の `featured_fish` / `allowed_fish` を使って分類してよい。分類が曖昧になったら、後続で魚データに `book_category` のような明示フィールドを追加する。

## 素材方針

釣り場マップと同じく、画面の第一印象を作る枠はPNG素材化する。テキストや数値はGodot側で描画する。

優先して用意する素材:

- 図鑑ヘッダー木枠
- ステータスバー
- 魚カード枠
- 選択中魚カード枠
- 未発見魚カード枠
- 右詳細パネル枠
- 詳細パネル内のラベルプレート
- フィルタボタン枠
- `港へ戻る` ボタン枠
- スクロールバー装飾

魚ポートレートは `assets/showcase/fish/*_card_portrait.png` を優先して使う。画面からは `FightFishAssets.card_portrait_path()` 経由で参照し、存在しない場合だけ、色付き簡易アイコンやシルエットにフォールバックする。

新規PNGは、`ResourceLoader.exists()` だけに依存せず、必要なら `ImageTexture.create_from_image()` のフォールバックを入れる。Godot import前のheadless確認でも表示が崩れないようにする。

## 実装手順

### 1. 入口と画面単位を決める

- `src/ui/fish_book_screen.gd` を新設する。
- `src/main.gd` に `fish_book` 画面IDを追加する。
- 港メニューの `ステータス・図鑑` を分離するか、ステータス画面から図鑑へ入る導線を追加する。
- 既存 `status_screen.gd` は、必要ならプレイヤーステータスと所持品中心に整理する。

### 2. 先に構成だけを合わせる

- 背景、上部ヘッダー、左グリッド、右詳細、下部フッターを配置する。
- この段階では細部の装飾より、完成イメージと同じ領域比率にする。
- 左55%、右35%、上部ヘッダー、下部フィルタの読み順を守る。

### 3. 魚カード一覧を実データ接続する

- `GameData.get_all_fish_ids()` でカードを作る。
- `PlayerProgress.caught_counts` で発見済み判定をする。
- `PlayerProgress.best_sizes` で最大サイズを出す。
- 魚ポートレートをロードする。
- 未発見魚は伏せカードにする。

### 4. 選択と右詳細を実装する

- 初期選択は最初の発見済み魚。未発見しかない場合はNo.001。
- カード押下で `_selected_fish_id` を更新する。
- 右詳細を再描画する。
- 選択中カードの視覚状態を更新する。

### 5. フィルタを実装する

- 下部ボタンで `_active_filter` を切り替える。
- グリッドの表示魚を更新する。
- 選択中魚がフィルタ外になった場合は、フィルタ内の先頭魚へ選択を移す。
- フィルタごとの件数や発見数を出すかは、表示が詰まらない範囲で決める。

### 6. PNG素材で質感を上げる

- 汎用パネルを木枠、羊皮紙、金具付きカードへ置き換える。
- テキストは焼き込まず、Godot側で読みやすく配置する。
- ボタンは標準Buttonの文字任せにせず、必要ならTextureRectとLabelを自前配置する。

### 7. 実スクショで調整する

- `/tmp/tsuri_fish_book.png` のような固定スクショを出す。
- `reference/07_fish_book_mockup.png` と並べて確認する。
- 文字の見切れ、余白の穴、カード密度、右詳細の読みやすさ、下部ボタンの主従を修正する。

### 8. 検証する

- Godot validateを通す。
- 図鑑画面のsmoke testを追加する。
- 港から図鑑へ入り、港へ戻れることを確認する。
- 未発見魚、発見済み魚、フィルタ切り替え、セーブデータあり/なしを確認する。

## 完了条件

最低限の完了条件:

- `fish_book` 画面として独立して開ける。
- 発見済み/未発見の表示差が成立している。
- 魚カード選択で右詳細が更新される。
- 下部フィルタが機能する。
- `港へ戻る` が機能する。
- `reference/07_fish_book_mockup.png` と同じ大枠の構成に見える。
- 1280x720のスクショで主要テキストが見切れない。
- Godot validateとsmoke testが通る。

80点品質の完了条件:

- 木製ヘッダー、羊皮紙カード、金縁、港背景の世界観が揃っている。
- 左グリッドと右詳細の視線誘導が明確。
- 選択中カードと主ボタンが一目で分かる。
- 未発見カードが安っぽい空欄ではなく、収集欲を出す伏せ表示になっている。
- 魚ポートレートが既存ファイト画面の素材とつながって見える。
- スクショ比較で、汎用UIから抜け出していると判断できる。

90点以上へ上げる場合の追加条件:

- 図鑑専用のカード枠、詳細枠、ボタン枠を本番素材として作る。
- 魚ごとの図鑑ポートレートの解像度と見せ方を揃える。
- `よく釣れる場所` に専用小サムネイルを使う。
- 見出し、数値、本文、ボタンでフォントの階層を作る。
- 魚の分類をデータ仕様として明示し、文字列推定に頼らない。

## 注意点

- `status_screen.gd` の既存図鑑を磨く方向に寄りすぎない。今回の完成イメージは独立図鑑画面である。
- 文字をPNGに焼き込まない。魚名、釣果数、最大サイズ、発見数はすべてデータで変わる。
- 魚カード数が30種類あるため、カードサイズとスクロールの安定性を先に確保する。
- 日本語は想定より横幅を使うため、実スクショで必ず確認する。
- 釣り場マップ実装と並行してもよいが、入口導線や港メニューを触るタイミングだけ衝突に注意する。
